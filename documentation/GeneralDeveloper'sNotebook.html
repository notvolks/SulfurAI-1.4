Installation

Install SulfurAI via the INSTALLER folder:

INSTALLER/INSTALL SULFURAI

Or run the automatic installer:

SulfurAI.py

Model Info + Creating / Identifying models.

SulfurAI uses multiple pre-trained transformer/classification models optimized for conversational tasks. Default model: sulfur_drl_[build].

Creating models isn't supported in build 10399d-patch9, but you can find current ones with the SulfurAI API.


(Assuming you imported SulfurAI as SulfurAI)

print(SulfurAI.retrieve_current_models())
print(SulfurAI.retrieve_active_model())
print(SulfurAI.retrieve_active_model_history())

Best practices when writing API code:



          
1. Use the SulfurAI import (don't abbrievate 'as')

          
2. Use custom SulfurAI tracebacks: [from scripts.ai_renderer_sentences.error import SulfurError] and [raise SulfurError(message=f"")]

          
3. Use the HollyFormat to structure your code [documentation/HollyFormat]

Directory Guide

Here is a brief overview of the SulfurAI directory (folders) structure:


     api - Hosts the API code and server cache.
    

     apps - contains appdata for external apps.
    

     data - contains data for the renderer & cache.
    

     documentation - the revamped developer SDK.
    

     extra_models - contains all ai models & related scripts used in Sulfur.
    

     installer - contains installer scripts for [apps].
    

     model_info - contains information about all models in Sulfur, see [extra_models].
    

     returns - the renders of all models, see [extra_models].
    

     scripts - contains ai render scripts which assist [extra_models].
    

     settings - contains sulfur settings.
    

     setup - contains files used in the startup or installation of Sulfur.

Bias Notice

⚠️ SulfurAI uses pre-trained data and may reflect biases present in the original dataset. Exercise caution for sensitive tasks and consider using custom models.
